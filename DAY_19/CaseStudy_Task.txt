1. Discovery
Decision
Use MCP tool discovery instead of hardcoding APIs.

Why
New Amazon capabilities can be added without changing agents.
Agents dynamically discover available tools.

2.Authorization
Decision

Separate credentials per MCP server.

Why

Principle of least privilege.

Human Approval Gates
Decision

Require approval before write actions.

Summary

| Domain         | Decision                          |
| -------------- | --------------------------------- |
| Discovery      | MCP tool discovery                |
| Authorization  | Per-server credentials            |
| Trust Boundary | Seller MCP vs Amazon MCP          |
| Reads          | Autonomous                        |
| Writes         | Human approval                    |
| PII            | Masked by default                 |
| Memory         | Store context, not sensitive data |
| Observability  | Log every MCP call                |
| Reliability    | Independent MCP servers           |
| Governance     | Approval gateway for money/PII    |
